# Security Policy

## Supported Versions

This project is currently in active development. Security fixes are applied to the latest `main` branch.

## Reporting a Vulnerability

Please do not open public issues for suspected vulnerabilities.

Use one of these private channels:

1. GitHub Security Advisories (preferred): "Report a vulnerability" in the repository Security tab.
2. If advisories are not available, contact a maintainer directly and include `[SECURITY]` in the subject.

Please include:

- Affected version/commit
- Reproduction steps or proof of concept
- Impact assessment
- Suggested mitigation (if available)

## Response Process

- Initial acknowledgement target: within 5 business days
- Triage and severity assessment after acknowledgement
- Fix development and coordinated disclosure once a patch is ready

We ask for responsible disclosure and reasonable time to investigate and fix issues before public disclosure.
